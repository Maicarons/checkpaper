# 功能特性

CheckPaper 提供 **6 种核心验证能力**，用于学术论文验证。每种类型可独立运行，也可组合使用进行全面分析。

## 1. 格式检查

**类型：** `format`

验证论文的结构和格式规范性。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 标题层级 | 验证标题级别一致且正确嵌套（H1 → H2 → H3） |
| 编号规范 | 检查章节、图表、公式编号是否连续无间断 |
| 字体一致性 | 验证相似元素的字体和字号保持统一 |
| 页面布局 | 检查页边距、页眉页脚、页面尺寸的合规性 |
| 目录准确性 | 验证目录与实际章节标题和页码对应 |
| 段落格式 | 验证缩进、行距和段落间距的一致性 |

### 严重程度

- **严重：** 标题层级跳过（如 H1 → H3 缺少 H2）
- **警告：** 编号间断或格式不一致
- **信息：** 不影响可读性的轻微样式偏差

## 2. 图表引用检查

**类型：** `figure_table`

确保所有图表在正文中被正确引用。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 引用完整性 | 每个定义的图表至少有一处正文引用 |
| 编号匹配 | 引用编号与实际定义编号一致 |
| 孤儿引用检测 | 发现引用了不存在的图表 |
| 未引用检测 | 发现定义了但未被引用的图表 |

### 示例问题

```
🔴 严重：第 8 页引用了 "Figure 5"，但只定义了 Figures 1-4
🟡 警告：第 5 页定义了 "Table 2"，但正文中从未引用
🔵 信息：建议使用 "如图所示" 格式以增加正式感
```

## 3. 参考文献引用检查

**类型：** `citation`

验证正文引用与参考文献列表之间的一致性。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 引用-文献匹配 | 每个 `[1]` 或 `(Author, Year)` 引用都有对应的参考文献条目 |
| 重复检测 | 识别参考文献列表中的重复条目 |
| 缺失引用 | 发现引用了不存在的文献条目 |
| 格式一致性 | 检查引用格式是否统一（全数字或全作者-年份） |
| 排序检查 | 验证参考文献按引用风格规则排序 |

## 4. 数据来源验证

**类型：** `data_source`

验证论文中引用的数据来源的真实性和可访问性。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 来源标注 | 数据来源是否明确引用（含 URL 或 DOI） |
| URL 可访问性 | 引用的数据集 URL 是否可访问 |
| 内容匹配 | 数据描述与实际数据集内容是否匹配 |
| 造假检测 | 识别可疑或伪造的数据来源声明 |

## 5. 数据处理验证

**类型：** `data_processing`

验证论文中的统计方法和数据处理声明。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| 统计方法 | 验证所声称的统计方法是否适合该数据 |
| 样本量 | 检查样本量是否足以支撑报告的分析 |
| p 值合理性 | 验证报告的 p 值和置信区间是否数学上合理 |
| GRIM/SPRITE 测试 | 执行 GRIM 和 SPRITE 测试检测不可能的统计量 |
| 图文一致性 | 检查图表中的数值是否与正文报告一致 |

### 统计测试

| 测试 | 用途 |
|------|------|
| **GRIM** | 测试报告的均值在给定样本量和量表粒度下是否可能 |
| **SPRITE** | 测试报告的统计量是否相互一致 |

## 6. 参考文献验证

**类型：** `reference`

通过外部学术数据库验证参考文献的真实性。

### 检查内容

| 检查项 | 说明 |
|--------|------|
| DOI 验证 | 验证 DOI 是否存在且正确解析 |
| 元数据匹配 | 将标题、作者、期刊、年份与 Crossref/Semantic Scholar 数据比对 |
| 期刊可信度 | 检测潜在的掠夺性或不存在的期刊 |
| 联网搜索 | 当 API 数据不足时通过网页搜索验证参考文献 |

### 使用的 API

| API | 用途 |
|-----|------|
| **Crossref** | DOI 元数据、引用次数、期刊信息 |
| **Semantic Scholar** | 论文元数据、引用图谱、影响力指标 |

## 组合验证类型

你可以对单篇论文运行任意组合的验证类型：

```bash
# 运行所有验证
curl -X POST http://localhost:9031/api/v1/validation/start \
  -H "Content-Type: application/json" \
  -d '{
    "document_id": "your-doc-id",
    "validation_types": ["format", "figure_table", "citation", "data_source", "data_processing", "reference"]
  }'

# 仅运行格式和引用检查（更快）
curl -X POST http://localhost:9031/api/v1/validation/start \
  -H "Content-Type: application/json" \
  -d '{
    "document_id": "your-doc-id",
    "validation_types": ["format", "citation"]
  }'
```

## 严重程度体系

所有问题按严重程度分类：

| 级别 | 图标 | 说明 | 示例 |
|------|------|------|------|
| **严重** | 🔴 | 可能表示造假或根本性错误的严重问题 | 伪造数据、缺失引用、不可能的统计量 |
| **警告** | 🟡 | 提交前应解决的问题 | 格式不一致、引用不完整 |
| **信息** | 🔵 | 改进建议 | 格式建议、优化提示 |
